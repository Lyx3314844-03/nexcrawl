# 安全与合规边界

> NexCrawl 的安全设计原则、明确不提供的能力、合规建议和生产部署规范

---

## 安全设计原则

NexCrawl 是一个**合规数据采集平台**，其安全设计基于以下原则：

1. **授权访问**: 只采集你有权访问的数据
2. **透明操作**: 所有操作记录到审计日志
3. **合规优先**: 遇到强风控时进入人机协作，而非自动绕过
4. **最小权限**: 使用最小权限账号和凭证
5. **租户隔离**: 多租户环境下的数据和凭证隔离

---

## 明确不提供的能力

### NexCrawl 不提供以下能力

| 类别 | 不提供的能力 |
| --- | --- |
| 访问控制绕过 | 绕过未经授权的访问控制 |
| 身份伪造 | 伪造身份、设备信誉或 Attestation Token |
| 设备验证绕过 | 绕过 Play Integrity、SafetyNet、DeviceCheck、App Attest |
| 验证码批量破解 | 批量破解验证码或账号风险页 |
| 未授权数据采集 | 未授权采集个人数据、付费内容或受保护系统 |

### 与"绕过"的区别

NexCrawl 的"智能"体现在：
- **识别**目标类型和阻断点
- **选择**合适的采集策略
- **指出**需要人工介入的环节
- **记录**完整操作审计

而不是无限制突破任何系统的安全措施。

## 强风控处理流程

### 检测信号

当检测到以下信号时，框架会触发合规流程：

| 信号 | 类型 |
| --- | --- |
| Play Integrity / SafetyNet | 设备 Attestation |
| DeviceCheck / App Attest | 苹果设备验证 |
| SSO (SAML/OIDC) | 企业单点登录 |
| 扫码登录 | 二维码认证 |
| Passkey / WebAuthn | 无密码认证 |
| 账号风险页 | 账号安全保护 |
| CAPTCHA | 验证码 |
| 速率限制 | 访问频率控制 |

### 框架行为

当检测到强风控信号时，框架会按以下步骤处理：

```
检测到强风控信号
       │
       ▼
┌──────────────────┐
│ 1. 生成合规计划   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 2. 阻断自动绕过   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 3. 进入人机协作   │
│    或人工审批     │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 4. 要求授权环境   │
│    (账号/设备)    │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 5. 记录审计日志   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 6. 生成合规建议   │
└──────────────────┘
```

## 合规建议

### 数据采集合规

| 建议 | 说明 |
| --- | --- |
| 只采集有权访问的数据 | 确认你拥有目标数据的合法采集权限 |
| 尊重 robots.txt | 遵守网站的爬虫协议 |
| 遵守服务条款 | 不违反目标网站的使用条款 |
| 遵守速率限制 | 不要对目标服务器造成过大负担 |
| 不绕过登录/付费墙 | 未经授权不访问受限内容 |
| 保护个人数据 | 遵守 GDPR、CCPA 等隐私法规 |

### 资源管理合规

| 建议 | 说明 |
| --- | --- |
| 租户隔离 | 为不同租户配置独立的账号池、代理池、设备池 |
| 凭证加密 | 启用 `NexCrawl_VAULT_KEY` 加密凭证 |
| 定期审计 | 检查 `.NexCrawl/audit.ndjson` 日志 |
| 最小权限 | 使用权限最小的服务账号 |

## 生产部署建议

### 网络安全

| 配置 | 说明 |
| --- | --- |
| API 密钥认证 | 配置 `NexCrawl_API_KEY` 环境变量 |
| TLS 加密 | 使用反向代理（Nginx/Caddy）添加 HTTPS |
| 防火墙规则 | 限制 API 端点的访问来源 |
| CORS 策略 | 配置允许的跨域来源 |

### 凭证管理

| 级别 | 方案 |
| --- | --- |
| 开发环境 | 本地 `.env` 文件（不提交到版本控制） |
| 测试环境 | `NexCrawl_VAULT_KEY` 加密落盘 |
| 生产环境 | 迁移到 Vault/KMS 等外部密钥管理系统 |

### 监控与告警

为以下指标配置告警：

| 指标 | 建议阈值 |
| --- | --- |
| 任务失败率 | > 10% |
| 登录态健康 | 失效账号 > 20% |
| 账号池健康 | 可用账号 < 最小阈值 |
| 设备池健康 | 在线设备 < 最小阈值 |
| 代理健康 | 失效代理 > 30% |
| API 延迟 | P99 > 5s |

### 部署架构

```
                    ┌─────────────┐
                    │   负载均衡   │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
       ┌──────▼──────┐ ┌──▼────────┐ ┌─▼──────────┐
       │  NexCrawl   │ │ NexCrawl  │ │  NexCrawl  │
       │  Instance 1 │ │ Instance 2│ │ Instance 3 │
       └──────┬──────┘ └──┬────────┘ └─┬──────────┘
              │            │            │
              └────────────┼────────────┘
                           │
                    ┌──────▼──────┐
                    │  共享存储    │

                    │   外部DB)   │
                    └─────────────┘
```

### 环境变量参考

| 变量 | 说明 | 必填 |
| --- | --- | :---: |
| `NexCrawl_API_KEY` | API 认证密钥 | 生产环境推荐 |
| `NexCrawl_VAULT_KEY` | 凭证加密密钥 | 生产环境推荐 |
| `NexCrawl_PORT` | 服务端口（默认 `3100`） | - |
| `NexCrawl_HOST` | 绑定地址（默认 `127.0.0.1`） | - |
| `NexCrawl_STATE_DIR` | 状态目录（默认 `.NexCrawl`） | - |

## 审计日志

### 日志位置

```
.NexCrawl/audit.ndjson
```

### 日志格式

每行一条 JSON 记录：

```json
{
  "timestamp": "2024-01-15T10:30:00.000Z",
  "tenantId": "tenant-a",
  "action": "job.create",
  "actor": "admin",
  "resource": "jobs",
  "result": "success",
  "details": {}
}
```

### 查看审计日志

```bash
# 通过 API 查询
curl http://127.0.0.1:3100/platform/governance/audit

# 或直接查看文件
cat .NexCrawl/audit.ndjson
```
