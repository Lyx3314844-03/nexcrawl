## 安全边界：明确不提供的能力

### 框架不会承诺

- 绕过 Play Integrity、SafetyNet、DeviceCheck、App Attest
- 伪造设备信誉或 Attestation Token
- 绕过未经授权的访问控制
- 自动解决所有验证码、账号风险、人工审核
- 保证任何目标都可采集

### 遇到强风控时的行为

当检测到以下信号时，框架会：

| 信号 | 框架行为 |
| --- | --- |
| Attestation 检测 | 停止自动化，生成合规计划 |
| 设备信誉检查 | 要求 owner-approved test device |
| SSO / 扫码 / Passkey | 进入人机 challenge |
| 账号风险页 | 人工审批或停止自动化 |
| 未授权访问控制 | 记录审计日志，生成合规说明 |

### 框架的升级路径

1. **停止自动化或降级**：不尝试绕过安全系统
2. **进入人机 challenge**：请求人工确认
3. **要求授权**：owner-approved test device / account
4. **记录审计**：完整记录事件到 `audit.ndjson`
5. **生成建议**：提供合规说明和下一步操作建议

## "万能"的实际含义

本项目里的"万能入口"指的是：

| 能力 | 说明 |
| --- | --- |
| 自动识别目标类型 | 分析 URL、HTML、响应头等判断页面类型 |
| 自动选择执行 Lane | 根据分析结果选择最佳采集策略 |
| 自动指出阻断点 | 明确告知哪些环节需要人工介入 |
| 人机协作 | 能进入人工确认或合规门禁流程 |
| 结果沉淀 | 把结果、事件、失败和修复建议完整记录 |

> **重要**：它不是无限制突破任何系统的工具，而是智能识别、合规采集、完整审计的数据采集平台。
